## Integration Patterns

### Option 1: Proxy (Recommended)

Wrap our tools in yours. Users only know about your platform.

**What you add:**
```typescript
yourplatform_file_dispute → proxies to our file_dispute
yourplatform_check_dispute → proxies to our check_case_status  
```

**Pros:** Seamless UX, single API key  
**Cons:** ~50 lines of proxy code per tool

---

### Option 2: Direct Reference

Tell users to add Consulate MCP server alongside yours.

**Your docs:**
```markdown
For disputes, also add Consulate:
{
  "mcpServers": {
    "yourplatform": {"url": "..."},
    "consulate": {"url": "https://api.consulatehq.com"}
  }
}
```

**Pros:** Zero code for you  
**Cons:** Users need 2 API keys

## Complete Proxy Example

```typescript
// In your MCP invocation handler
export const mcpInvoke = async (ctx, request) => {
  const { tool, parameters } = await request.json();
  
  // Your native tools
  if (tool === 'yourplatform_process_payment') {
    return await handlePayment(ctx, parameters);
  }
  
  // Proxied dispute tool
  if (tool === 'yourplatform_file_dispute') {
    // Get context from your DB
    const txn = await ctx.db.get(parameters.transactionId);
    
    // Forward to Consulate
    const response = await fetch('https://api.consulatehq.com/mcp/invoke', {
      method: 'POST',
      headers: {
        'X-Agent-DID': PARTNER_AGENT_DID,
        'X-Signature': signatureHex,
        'X-Timestamp': timestamp.toString(),
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        tool: 'file_dispute',
        parameters: {
          plaintiff: txn.buyerDid,
          defendant: txn.sellerDid,
          disputeType: 'PAYMENT_FAILURE',
          claim: parameters.reason,
          claimAmount: parameters.amount
        }
      })
    });
    
    const result = await response.json();
    
    // Save case ID in your DB
    await ctx.db.patch(txn._id, {
      disputeId: result.caseId,
      disputeStatus: 'filed'
    });
    
    return new Response(JSON.stringify({
      disputeId: result.caseId,
      trackingUrl: `https://yourplatform.com/disputes/${result.caseId}`,
      estimatedResolution: result.estimatedResolution
    }));
  }
};
```

## Available Tools to Proxy

| Our Tool | What It Does | Suggested Wrapper Name |
|----------|-------------|----------------------|
| `file_dispute` | File new dispute | `yourplatform_file_dispute` |
| `check_case_status` | Get case status | `yourplatform_check_dispute` |
| `submit_evidence` | Upload evidence | `yourplatform_submit_evidence` |
| `list_my_cases` | List disputes | Optional |
| `get_sla_status` | Check compliance | Optional |

**Skip these** (platform-agnostic):
- `register_agent` - Users register directly
- `lookup_agent` - Not needed if you know both parties
- `request_vendor_registration` - Users handle directly

## Testing

```bash
# 1. Test our MCP server
curl https://api.consulatehq.com/.well-known/mcp.json | jq '.tools[].name'

# 2. Test auth (should fail without key)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "Content-Type: application/json" \
  -d '{"tool":"lookup_agent","parameters":{"query":"test"}}'
# Expected: 401

# 3. Test with partner key (should succeed)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "X-Agent-DID: did:agent:partner-123" \
  -H "X-Signature: <signature>" \
  -H "X-Timestamp: 1234567890123" \
  -H "Content-Type: application/json" \
  -d '{"tool":"lookup_agent","parameters":{"query":"OpenAI"}}'
# Expected: 200 with results
```
