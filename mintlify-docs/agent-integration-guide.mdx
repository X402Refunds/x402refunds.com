</Note>

## Quick Start

<Steps>
  <Step title="Install the SDK">
    Install the Consulate agent SDK via npm:
    
    ```bash
    npm install @consulate/agent-sdk
    ```
    
    <Tip>
    The SDK handles authentication, dispute filing, evidence submission, and real-time notifications automatically.
    </Tip>
  </Step>

  <Step title="Register Your Agent">
    Create and register your agent with Consulate:
    
    ```javascript
    import { ConsulateAgent } from '@consulate/agent-sdk';
    
    const agent = new ConsulateAgent({
      did: "did:agent:your-service-api",
      ownerDid: "did:enterprise:yourcompany",
      consulateUrl: "https://api.consulatehq.com"
    });
    
    await agent.register({
      functionalType: "api",
      capabilities: ["payment-processing", "data-enrichment"],
      stake: 50000
    });
    ```
    
    <Info>
    **Decentralized Identifiers (DIDs):** Every agent needs a unique DID. If you don't have one, use the format: `did:agent:your-service-name`
    </Info>
  </Step>

  <Step title="Set Up Monitoring">
    Configure SLA monitoring and dispute handlers:
    
    ```javascript
    // Set up SLA monitoring
    agent.setSLACollector(async () => ({
      availability: 99.9,
      responseTime: 200,
      errorRate: 0.1
    }));
    
    // Handle disputes automatically
    agent.onDispute('DISPUTE_FILED', async (notification) => {
      console.log('Dispute filed:', notification);
      // Your custom dispute handling logic
    });
    
    // Start automated monitoring
    await agent.startMonitoring();
    ```
    
    <Check>
    Your agent is now monitoring SLAs and ready to handle disputes automatically!
    </Check>
  </Step>
</Steps>

## Integration Methods

<Tabs>
  <Tab title="SDK (Recommended)">
    ### JavaScript/TypeScript SDK
    
    The official SDK provides the simplest integration path:
    
    ```bash
    npm install @consulate/agent-sdk
    ```
    
    **Features:**
    - Automatic authentication
    - Type-safe API calls
    - Built-in retry logic
    - Real-time event handlers
    - Dispute lifecycle management
    
    **Best for:**
    - Node.js applications
    - TypeScript projects
    - Quick integration
  </Tab>
  
  <Tab title="HTTP API">
    ### Direct HTTP API Integration
    
    Use our REST API for maximum flexibility:
    
    ```bash
    curl -X POST https://api.consulatehq.com/agents/register \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "did": "did:agent:your-service",
        "name": "Your Service API",
        "functionalType": "api"
      }'
    ```
    
    **Features:**
    - Language agnostic
    - Complete control
    - No dependencies
    
    **Best for:**
    - Non-JavaScript environments
    - Custom implementations
    - Microservices
    
    See [API Reference](/api-endpoints) for complete documentation.
  </Tab>
  
  <Tab title="MCP Integration">
    ### Model Context Protocol
    
    Integrate via the Model Context Protocol for AI-native applications:
    
    ```json
    {
      "mcpServers": {
        "Consulate": {
          "url": "https://api.consulatehq.com/mcp"
        }
      }
    }
    ```
    
    **Features:**
    - AI-native integration
    - Context-aware dispute resolution
    - Works with Claude Desktop, Cursor, etc.
    
    **Best for:**
    - AI agent frameworks
    - Claude Desktop users
    - Interactive dispute resolution
    
    Learn more in the [Integration Strategy guide](/agent-integration-strategy).
  </Tab>
</Tabs>

## Core API Endpoints

<AccordionGroup>
  <Accordion title="Agent Management" icon="robot">
    **Register Agent**
    ```
    POST /agents/register
    ```
    Register your agent with Consulate
    
    **Discover Agents**
    ```
    GET /agents/discover
    ```
    Find other registered agents by DID
    
    **Get Capabilities**
    ```
    GET /agents/capabilities
    ```
    View agent capabilities and status
  </Accordion>
  
  <Accordion title="SLA Monitoring" icon="chart-line">
    **Report SLA Metrics**
    ```
    POST /sla/report
    ```
    Submit SLA performance metrics
    
    **Get SLA Status**
    ```
    GET /sla/status/:agentDid
    ```
    Check current SLA compliance status
  </Accordion>
  
  <Accordion title="Dispute Management" icon="scale-balanced">
    **File Dispute**
    ```
    POST /disputes/file
    ```
    Initiate a new dispute case
    
    **Submit Evidence**
    ```
    POST /evidence/submit
    ```
    Upload evidence for a dispute
    
    **Check Case Status**
    ```
    GET /disputes/:disputeId/status
    ```
    Monitor dispute resolution progress
  </Accordion>
  
  <Accordion title="Notifications" icon="bell">
    **Register Webhook**
    ```
    POST /webhooks/register
    ```

    **Get Notifications**

    ```
    GET /live/feed
    ```

  </Accordion>
</AccordionGroup>

## Authentication

<Warning>
All API requests require authentication using an API key in the `Authorization` header.
</Warning>

### Generate an API Key

<Steps>
  <Step title="Log in to Dashboard">
    Visit [consulatehq.com/dashboard](https://consulatehq.com/dashboard)
  </Step>
  
  <Step title="Navigate to Settings">
    Go to Settings → API Keys
  </Step>
  
  <Step title="Create New Key">
    Click "Create New API Key" and copy the generated key
  </Step>
  
  <Step title="Use in Requests">
    Include in all API requests:
    ```bash
    Authorization: Bearer YOUR_API_KEY
    ```
  </Step>
</Steps>

### Example Authenticated Request

```javascript
const response = await fetch('https://api.consulatehq.com/agents/register', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${YOUR_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    did: 'did:agent:your-service',
    name: 'Your Service API',
    functionalType: 'api'
  })
});
```
