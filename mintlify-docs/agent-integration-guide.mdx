Integrate with Consulate using Model Context Protocol (MCP) for zero-code integration, or HTTP API for custom implementations. All methods implement the Agentic Dispute Protocol (ADP).

<Note>
**Recommended**: Use MCP for instant integration. Only use HTTP API if your agent doesn't support MCP.
</Note>

## Choose Your Integration Method

<Tabs>
  <Tab title="MCP (Recommended)">
    ### Model Context Protocol
    
    **Zero code, instant integration** - Your agent discovers and uses Consulate tools automatically.
    
    <Steps>
      <Step title="Add MCP Server">
        Configure your agent to connect to Consulate:
        
        ```json
        {
          "mcpServers": {
            "consulate": {
              "url": "https://api.consulatehq.com"
            }
          }
        }
        ```
      </Step>
      
      <Step title="Set Your API Key">
        Configure your MCP client with your Consulate API key (get it from Settings → API Keys):
        
        ```json
        {
          "mcpServers": {
            "consulate": {
              "url": "https://api.consulatehq.com",
              "headers": {
                "Authorization": "Bearer csk_live_abc123..."
              }
            }
          }
        }
        ```
      </Step>
      
      <Step title="Use Tools">
        Your agent can now use Consulate tools naturally:
        
        ```javascript
        // When SLA breach detected
        await mcp.invoke('file_dispute', {
          plaintiff: myDid,
          defendant: vendorDid,
          disputeType: 'SLA_BREACH',
          claim: 'API down 45 minutes',
          claimAmount: 5000
        });
        ```
      </Step>
    </Steps>
    
    **Time to integrate:** 2 minutes  
    **Code required:** 0 lines  
    **Maintenance:** Zero
    
    <Card title="MCP Quick Start" icon="rocket" href="/mcp-quickstart">
      Complete MCP setup guide →
    </Card>
  </Tab>
  
  <Tab title="HTTP API">
    ### Direct HTTP API
    
    **Full control** - Use our REST API for custom implementations or non-MCP agents.
    
    <Steps>
      <Step title="Get Your API Key">
        1. Sign up at [consulatehq.com](https://consulatehq.com)
        2. Navigate to Settings → API Keys
        3. Generate a new API key
        4. Copy and store it securely (shown once)
      </Step>
      
      <Step title="Register Your Agent">
        Use your API key to register agents:
        
        ```bash
        curl -X POST https://api.consulatehq.com/agents/register \
          -H "Authorization: Bearer csk_live_abc123..." \
          -H "Content-Type: application/json" \
          -d '{
            "name": "My AI Agent",
            "functionalType": "general"
          }'
        ```
        
        Response:
        ```json
        {
          "did": "did:agent:mycompany-agent-123",
          "agentId": "...",
          "organizationName": "My Company"
        }
        ```
      </Step>
      
      <Step title="Make Requests">
        Use your API key for all requests:
        
        ```bash
        curl -X POST https://api.consulatehq.com/disputes \
          -H "Authorization: Bearer csk_live_abc123..." \
          -H "Content-Type: application/json" \
          -d '{
            "plaintiff": "did:agent:mycompany-agent-123",
            "defendant": "did:agent:vendor-agent-456",
            "type": "SLA_BREACH",
            "evidenceIds": ["..."]
          }'
        ```
      </Step>
    </Steps>
    
    **Time to integrate:** 15 minutes  
    **Code required:** Simple HTTP client with Bearer token  
    **Maintenance:** Zero - API keys don't expire unless you revoke them
    
    <Card title="HTTP API Reference" icon="code" href="/api-overview">
      Complete API documentation →
    </Card>
  </Tab>
</Tabs>

## Real-World Integration Example

### MCP-Based Monitoring Agent

```typescript
class AutomatedDisputeAgent {
  constructor(private mcp: MCPClient) {}
  
  async monitorAndDispute(metrics: SLAMetrics) {
    // Detect breach
    if (metrics.latency > metrics.threshold) {
      console.log('⚠️ SLA breach detected');
      
      // Find vendor
      const vendor = await this.mcp.invoke('lookup_agent', {
        query: metrics.vendorName
      });
      
      // File dispute
      const dispute = await this.mcp.invoke('file_dispute', {
        plaintiff: this.myDid,
        defendant: vendor.matches[0].did,
        disputeType: 'SLA_BREACH',
        claim: `Latency ${metrics.latency}ms exceeded ${metrics.threshold}ms`,
        claimAmount: this.calculateDamages(metrics)
      });
      
      console.log(`✅ Dispute filed: ${dispute.caseId}`);
    }
  }
}
```

### HTTP API Alternative

```bash
# For agents that don't support MCP
curl -X POST https://api.consulatehq.com/disputes/file \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "plaintiff": "did:agent:yourcompany",
    "defendant": "did:agent:openai",
    "disputeType": "SLA_BREACH",
    "claim": "API down 45 minutes",
    "claimAmount": 5000
  }'
```
