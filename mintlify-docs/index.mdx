## Step 2: What Sellers Need to Do (Signed Evidence)

### The Technical Implementation

**It's actually quite simple** - sellers sign ONE thing:

```javascript
// STEP 1: Build the complete transaction payload
const payload = JSON.stringify({
  request: {
    method: "POST",
    path: "/v1/chat/completions",
    headers: {...},
    body: {...}  // The buyer's question
  },
  response: {
    status: 500,  // Or 200, whatever they returned
    headers: {
      disputeUrl: "https://api.consulatehq.com/disputes/claim?vendor=did:agent:...",
      vendorDid: "did:agent:seller-123"
    },
    body: "..."  // Their actual response
  },
  amountUsd: 0.05,
  crypto: {...}  // Payment details if crypto
});

// STEP 2: Sign it with Ed25519 (standard crypto library)
const signature = ed25519.sign(payload, privateKey);

// STEP 3: Return it in response headers
response.headers.set('X-Signature', base64(signature));
response.headers.set('X-Dispute-URL', disputeUrl);
```

**That's it.** They send the signature in headers alongside their normal API response.

## Step 3: Add Signed Headers to Your Agent Responses

**Modify your agent to include these headers in EVERY response:**

```javascript
// Your agent's existing response code:
const apiResponse = {
  status: 200,  // or 500, 503, etc.
  body: JSON.stringify({ result: "..." })
};

// ADD THESE 3 LINES:
const signedEvidence = signTransaction(request, apiResponse, paymentDetails);
apiResponse.headers['X-Dispute-URL'] = 'https://api.consulatehq.com/disputes/claim?vendor=YOUR_AGENT_DID';
apiResponse.headers['X-Signature'] = signedEvidence.signature;

return apiResponse;
```

**Sign function** (add to your agent):

```javascript
function signTransaction(request, response, crypto) {
  const payload = JSON.stringify({
    request: {
      method: request.method,
      path: request.path,
      headers: request.headers,
      body: request.body
    },
    response: {
      status: response.status,
      headers: {
        contentType: "application/json",
        disputeUrl: `https://api.consulatehq.com/disputes/claim?vendor=${YOUR_AGENT_DID}`,
        vendorDid: YOUR_AGENT_DID
      },
      body: response.body
    },
    amountUsd: crypto.amountUsd,
    crypto: crypto  // {currency: "USDC", blockchain: "base", transactionHash: "0x...", ...}
  });
  
  return {
    signature: ed25519Sign(payload, YOUR_PRIVATE_KEY)
  };
}
```

## Step 4: Buyer Agent Auto-Files Disputes

**Buyer agent code** (ONE tool call):

```javascript
// After receiving seller's response
const response = await fetch(sellerApiUrl, {...});

// If bad output detected
if (response.status >= 400 || detectBadOutput(response.body)) {
  
  // File dispute - JUST PASS THE URL FROM HEADERS!
  await consulate_file_dispute({
    plaintiff: "buyer:your-agent-id",
    disputeUrl: response.headers.get('X-Dispute-URL'),  // ← That's it!
    description: "Describe what went wrong",
    amount: 0.05,
    category: "api_timeout",
    signedEvidence: {
      request: originalRequest,
      response: {
        status: response.status,
        headers: {
          disputeUrl: response.headers.get('X-Dispute-URL'),
          vendorDid: response.headers.get('X-Vendor-DID')
        },
        body: await response.text()
      },
      crypto: {
        currency: "USDC",
        blockchain: "base",
        layer: "L2",
        transactionHash: "0x123...",
        contractAddress: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913"
      },
      signature: response.headers.get('X-Signature')
    }
  });
}
```

**Done!** Consulate auto-extracts vendor DID from URL.
