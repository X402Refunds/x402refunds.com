## Quick Integration

- **`file_dispute`** - File payment disputes automatically
- **`check_case_status`** - Track dispute resolution

        <Accordion title="📋 Copy-Paste Prompt for Cursor/Windsurf">
      ```
      Create an MCP proxy server with two endpoints:
      
      1. GET /.well-known/mcp.json
         - Fetch from https://api.consulatehq.com/.well-known/mcp.json
         - Filter tools array to only: consulate_file_dispute, consulate_check_case_status
         - Strip 'consulate_' prefix from tool names
         - Return the modified schema
      
      2. POST /mcp/invoke
         - Accept {tool, parameters} in request body
         - Add 'consulate_' prefix to tool name
         - Forward to https://api.consulatehq.com/mcp/invoke
         - Add Authorization header: Bearer ${process.env.CONSULATE_API_KEY}
         - Return response as JSON
      
      Use Express.js with proper error handling.
      ```
        </Accordion>

        **Manual implementation:**

        ```javascript
const express = require('express');
const app = express();

app.use(express.json());

// Auto-expose Consulate tools with clean names
app.get('/.well-known/mcp.json', async (req, res) => {
  try {
    const response = await fetch('https://api.consulatehq.com/.well-known/mcp.json');
    const schema = await response.json();
    
    // Filter to core tools only
    const CORE_TOOLS = ['consulate_file_dispute', 'consulate_check_case_status'];
    
    const tools = schema.tools
      .filter(t => CORE_TOOLS.includes(t.name))
      .map(t => ({
        ...t,
        name: t.name.replace('consulate_', '') // Clean names
      }));
    
    res.json({ ...schema, tools });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// Proxy tool calls to Consulate
app.post('/mcp/invoke', async (req, res) => {
  try {
    const { tool, parameters } = req.body;
    
    const response = await fetch('https://api.consulatehq.com/mcp/invoke', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.CONSULATE_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        tool: 'consulate_' + tool, // Add prefix back
        parameters
      })
    });
    
    const result = await response.json();
    res.json(result);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(3000, () => {
  console.log('MCP proxy running on port 3000');
});
        ```

      <Tab title="Python">
        <Accordion title="📋 Copy-Paste Prompt for Cursor/Windsurf">
      ```
      Create an MCP proxy server using FastAPI with two endpoints:
      
      1. GET /.well-known/mcp.json
         - Fetch from https://api.consulatehq.com/.well-known/mcp.json using httpx
         - Filter tools array to only: consulate_file_dispute, consulate_check_case_status
         - Strip 'consulate_' prefix from tool names
         - Return the modified schema
      
      2. POST /mcp/invoke
         - Accept {tool: str, parameters: dict} in request body
         - Add 'consulate_' prefix to tool name
         - Forward to https://api.consulatehq.com/mcp/invoke
         - Add Authorization header: Bearer {CONSULATE_API_KEY from env}
         - Return response as JSON
      
      Use FastAPI with proper async/await and error handling.
      ```
        </Accordion>

        **Manual implementation:**

        ```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import httpx
import os

app = FastAPI()

CONSULATE_API_KEY = os.getenv('CONSULATE_API_KEY')
CONSULATE_BASE_URL = 'https://api.consulatehq.com'
CORE_TOOLS = ['consulate_file_dispute', 'consulate_check_case_status']

@app.get('/.well-known/mcp.json')
async def get_mcp_schema():
    """Auto-expose Consulate tools with clean names"""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.get(f'{CONSULATE_BASE_URL}/.well-known/mcp.json')
            response.raise_for_status()
            schema = response.json()
        
        # Filter to core tools and strip prefix
        tools = [
            {**tool, 'name': tool['name'].replace('consulate_', '')}
            for tool in schema['tools']
            if tool['name'] in CORE_TOOLS
        ]
        
        return {**schema, 'tools': tools}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

class MCPInvoke(BaseModel):
    tool: str
    parameters: dict

@app.post('/mcp/invoke')
async def invoke_tool(request: MCPInvoke):
    """Proxy tool calls to Consulate"""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f'{CONSULATE_BASE_URL}/mcp/invoke',
                json={
                    'tool': f'consulate_{request.tool}',  # Add prefix back
                    'parameters': request.parameters
                },
                headers={
                    'Authorization': f'Bearer {CONSULATE_API_KEY}',
                    'Content-Type': 'application/json'
                }
            )
            response.raise_for_status()
            return response.json()
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

if __name__ == '__main__':
    import uvicorn
    uvicorn.run(app, host='0.0.0.0', port=3000)
        ```

## What Your Agent Gets

Your agent automatically discovers these tools with full schemas:

<AccordionGroup>
  <Accordion title="file_dispute - File payment disputes" icon="scale-balanced">
    **Auto-discovered schema includes:**
    - All parameter types and descriptions
    - Required vs optional fields
    - Enum values for `disputeReason`
    - Pricing tier information

    **Example agent usage:**
    ```javascript
    // Agent calls this automatically when payment fails
    {
      "transactionId": "txn_abc123",
      "amount": 10.50,
      "plaintiff": "customer@example.com",
      "defendant": "merchant@example.com",
      "disputeReason": "service_not_rendered",
      "description": "Charged but API call failed"
    }
    ```

    **Dispute reasons:**
    - `service_not_rendered` - Paid but didn't receive service
    - `api_timeout` - API timed out but charge went through
    - `amount_incorrect` - Wrong amount charged
    - `duplicate_charge` - Charged multiple times
    - `fraud` - Unauthorized transaction
  </Accordion>

  <Accordion title="check_case_status - Track dispute progress" icon="magnifying-glass">
    **Auto-discovered schema includes:**
    - Case ID parameter (required)
    - Response format with status, timeline, ruling

    **Example agent usage:**
    ```javascript
    {
      "caseId": "case_abc123"
    }
    ```

    **Response includes:**
    - Current status (FILED, UNDER_REVIEW, DECIDED)
    - Estimated resolution time
    - Tracking URL
    - Ruling (when decided)
  </Accordion>
</AccordionGroup>

## Advanced: Add More Tools

Want to expose more than 2 tools? Update the filter:

```javascript
const CORE_TOOLS = [
  'consulate_file_dispute',
  'consulate_check_case_status',
  'consulate_submit_evidence', // Add evidence after filing
  'consulate_list_my_cases'     // List all disputes
];
```

Or expose all tools:

```javascript
// Remove filter - expose everything
const tools = schema.tools.map(t => ({
  ...t,
  name: t.name.replace('consulate_', '')
}));
```
