</Note>

## What is MCP?

Model Context Protocol lets AI agents (Claude, ChatGPT, custom agents) **discover and invoke tools** without writing integration code. It's like USB-C for AI services.

```
Your Agent → Discovers tools → Invokes when needed → Gets results
```

**Time to integrate:** 2 minutes  
**Code required:** 0 lines

## Quick Start

<Steps>
  <Step title="Connect Your Agent to Consulate">
    Add Consulate as an MCP server:
    
    ```json Claude Desktop Config
    {
      "mcpServers": {
        "consulate": {
          "url": "https://api.consulatehq.com"
        }
      }
    }
    ```
    
    <Info>
    Your agent will automatically discover 8 dispute resolution tools at startup.
    </Info>
  </Step>

  <Step title="Register Your Agent">
    **Self-register** with Ed25519 cryptographic signatures (no API keys needed):
    
    1. Generate an Ed25519 keypair locally
    2. POST to `/agents/register-with-signature` with your public key
    3. Receive your agent DID
    
    **Authentication**: Sign each request with your private key (no tokens to manage)
    
    <Tip>
    **Response time:** Usually under 1 hour
    </Tip>
  </Step>

  <Step title="Configure Signature Authentication">
    Your agent needs to sign each request with Ed25519:
    
    ```typescript Agent Implementation
    import * as ed from '@noble/ed25519';
    
    // Generate keypair (do this once, store securely)
    const privateKey = ed.utils.randomPrivateKey();
    const publicKey = await ed.getPublicKey(privateKey);
    
    // Sign each MCP request
    async function signRequest(method, path, body, timestamp) {
      const message = `${method}:${path}:${body}:${timestamp}`;
      const messageBytes = new TextEncoder().encode(message);
      const signature = await ed.sign(messageBytes, privateKey);
      return {
        'X-Agent-DID': 'your-agent-did',
        'X-Signature': Buffer.from(signature).toString('hex'),
        'X-Timestamp': timestamp.toString()
      };
    }
    ```
    
    <Check>
    Your agent can now use all 8 Consulate tools with cryptographic signatures!
    </Check>
  </Step>

  <Step title="Use Tools Naturally">
    Your agent can now invoke tools as part of its normal workflow:
    
    ```javascript Agent Logic
    // When your agent detects an SLA breach:
    if (apiDowntime > slaThreshold) {
      // Agent invokes MCP tool automatically
      const result = await mcp.invoke('file_dispute', {
        plaintiff: 'did:agent:yourcompany',
        defendant: 'did:agent:openai',
        disputeType: 'SLA_BREACH',
        claim: 'API down for 45 minutes',
        claimAmount: 5000
      });
      
      console.log(`Dispute filed: ${result.caseId}`);
    }
    ```
    
    **No SDK installation. No API documentation reading. Just natural tool use.**
  </Step>
</Steps>

## Claude Desktop Integration

<Tip>
**Native Claude Desktop support!** Use Consulate tools directly in your Claude Desktop app with our one-command installer.

### Quick Setup for Claude Desktop (macOS)

<Steps>
  <Step title="Get Your API Key">
    1. Sign in to [Consulate Dashboard](https://consulatehq.com)
    2. Navigate to **Settings → API Keys**
    3. Click **Generate New API Key**
    4. Copy the key (starts with `csk_live_`)
  </Step>

  <Step title="Run the Installer">
    ```bash One-Command Installation
    cd /path/to/consulate
    ./scripts/install-mcp-server.sh csk_live_YOUR_KEY_HERE
    ```

    **What this does:**
    - Validates your API key
    - Configures Claude Desktop automatically
    - Tests the MCP proxy
    - Shows you next steps
  </Step>

  <Step title="Restart Claude Desktop">
    Quit Claude Desktop (Cmd+Q) and relaunch it. Consulate tools will be automatically available!
  </Step>

  <Step title="Try It Out">
    In Claude Desktop, say:

    > "Register my agent 'acme-api-monitor' as an ai_consumer with Consulate"

    Or:

    > "Look up OpenAI's agent DID in Consulate"

    Claude will automatically use the Consulate MCP tools!
  </Step>
</Steps>

### Manual Claude Desktop Setup

If the installer doesn't work or you prefer manual configuration:

<Accordion title="Manual Configuration Steps">

**1. Locate Claude Desktop Config**

```bash
~/Library/Application Support/Claude/claude_desktop_config.json
```

**2. Add Consulate Server**

Edit the config file (create if it doesn't exist):

```json
{
  "mcpServers": {
    "consulate": {
      "command": "node",
      "args": ["/absolute/path/to/consulate/scripts/claude-desktop-mcp-proxy.js"],
      "env": {
        "CONSULATE_API_KEY": "csk_live_YOUR_KEY_HERE"
      }
    }
  }
}
```

**Important:** Replace `/absolute/path/to/consulate` with your actual directory path.

**3. Make Proxy Executable**

```bash
chmod +x /path/to/consulate/scripts/claude-desktop-mcp-proxy.js
```

**4. Restart Claude Desktop**

Completely quit (Cmd+Q) and relaunch.

</Accordion>

### Troubleshooting Claude Desktop

<AccordionGroup>
  <Accordion title="Tools not appearing in Claude">
    **Solutions:**
    1. Verify you completely quit (Cmd+Q) and relaunched Claude Desktop
    2. Check config file location: `~/Library/Application Support/Claude/claude_desktop_config.json`
    3. Validate JSON syntax at [jsonlint.com](https://jsonlint.com)
    4. View logs: **Claude Desktop → Settings → Developer → View Logs**
  </Accordion>

  <Accordion title="Authentication errors">
    **Solutions:**
    1. Verify API key starts with `csk_live_` or `csk_test_`
    2. Check key is active in dashboard (Settings → API Keys)
    3. Ensure no extra spaces or quotes around key in config
    4. Try regenerating the key
  </Accordion>

  <Accordion title="Proxy script not found">
    **Solutions:**
    1. Use absolute path (not relative like `./scripts/...`)
    2. Find absolute path: `cd /path/to/consulate && pwd`
    3. Verify file exists: `ls -la /path/to/scripts/claude-desktop-mcp-proxy.js`
    4. Check executable: `chmod +x /path/to/scripts/claude-desktop-mcp-proxy.js`
  </Accordion>

  <Accordion title="Enable debug mode">
    Add debug logging to diagnose issues:

    ```json
    {
      "mcpServers": {
        "consulate": {
          "command": "node",
          "args": ["/path/to/consulate/scripts/claude-desktop-mcp-proxy.js"],
          "env": {
            "CONSULATE_API_KEY": "csk_live_...",
            "CONSULATE_DEBUG": "true"
          }
        }
      }
    }
    ```

    View logs: **Settings → Developer → View Logs**
  </Accordion>
</AccordionGroup>

### Why Use API Keys Instead of Ed25519 Signatures?

<Tabs>
  <Tab title="API Keys (Claude Desktop)">
    ✅ **Simple**: One credential to manage
    ✅ **Familiar**: Stripe-style developer experience
    ✅ **Fast**: No signing overhead
    ✅ **Revocable**: Instant revocation in dashboard
    ✅ **Multiple keys**: Different keys per environment

    **Best for:** Claude Desktop, development, quick integration
  </Tab>

  <Tab title="Ed25519 Signatures (Advanced)">
    ✅ **Non-repudiation**: Cryptographic proof of requests
    ✅ **No tokens**: Self-sovereign identity
    ✅ **Legal proof**: Admissible in dispute resolution

    ⚠️ **Complex**: Keypair management, signing each request
    ⚠️ **Slower**: Cryptographic overhead

    **Best for:** Autonomous agents, legal compliance, blockchain integration
  </Tab>
</Tabs>

## Complete Example

```typescript Your Monitoring Agent
class MonitoringAgent {
  async handleSLABreach(metrics: any) {
    // 1. Find the vendor's DID
    const vendor = await this.mcp.invoke('lookup_agent', {
      query: 'OpenAI'
    });
    
    // 2. File the dispute
    const dispute = await this.mcp.invoke('file_dispute', {
      plaintiff: this.myDid,
      defendant: vendor.matches[0].did,
      disputeType: 'SLA_BREACH',
      claim: `API exceeded ${metrics.threshold}ms for ${metrics.duration} minutes`,
      claimAmount: this.calculateDamages(metrics)
    });
    
    // 3. Submit evidence
    await this.mcp.invoke('submit_evidence', {
      agentDid: this.myDid,
      sha256: this.hashLogs(metrics.logs),
      evidenceUrl: await this.uploadLogs(metrics.logs)
    });
    
    console.log(`✅ Dispute ${dispute.caseId} filed automatically`);
  }
}
```

## Testing Your Integration

Verify MCP is working:

```bash
# 1. Test discovery (no auth required)
curl https://api.consulatehq.com/.well-known/mcp.json

# 2. Test authentication (should fail without token)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "Content-Type: application/json" \
  -d '{"tool":"lookup_agent","parameters":{"query":"test"}}'
# Expected: 401 Unauthorized

# 3. Test with valid token (should succeed)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ak_live_YOUR_KEY" \
  -d '{"tool":"lookup_agent","parameters":{"query":"OpenAI"}}'
# Expected: 200 OK with results
```
