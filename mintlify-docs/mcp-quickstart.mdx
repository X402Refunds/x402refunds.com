<Note>
**Zero code required.** Your AI agent can discover and use Consulate's dispute resolution tools automatically through the Model Context Protocol (MCP).
</Note>

## What is MCP?

Model Context Protocol lets AI agents (Claude, ChatGPT, custom agents) **discover and invoke tools** without writing integration code. It's like USB-C for AI services.

```
Your Agent → Discovers tools → Invokes when needed → Gets results
```

**Time to integrate:** 2 minutes  
**Code required:** 0 lines

## Quick Start

<Steps>
  <Step title="Connect Your Agent to Consulate">
    Add Consulate as an MCP server:
    
    ```json Claude Desktop Config
    {
      "mcpServers": {
        "consulate": {
          "url": "https://api.consulatehq.com"
        }
      }
    }
    ```
    
    <Info>
    Your agent will automatically discover 8 dispute resolution tools at startup.
    </Info>
  </Step>

  <Step title="Get Your API Key">
    **Contact us** at support@consulatehq.com with:
    - Your company name
    - Agent use case (monitoring, integration, etc.)
    
    We'll send you:
    - Your agent DID
    - API key for authentication
    
    <Tip>
    **Response time:** Usually under 1 hour
    </Tip>
  </Step>

  <Step title="Add Authentication">
    Configure your MCP client to use Bearer token authentication:
    
    ```json Claude Desktop Config (Updated)
    {
      "mcpServers": {
        "consulate": {
          "url": "https://api.consulatehq.com",
          "headers": {
            "Authorization": "Bearer ak_live_YOUR_RANDOM_KEY"
          }
        }
      }
    }
    ```
    
    <Check>
    Your agent can now use all 8 Consulate tools!
    </Check>
  </Step>

  <Step title="Use Tools Naturally">
    Your agent can now invoke tools as part of its normal workflow:
    
    ```javascript Agent Logic
    // When your agent detects an SLA breach:
    if (apiDowntime > slaThreshold) {
      // Agent invokes MCP tool automatically
      const result = await mcp.invoke('file_dispute', {
        plaintiff: 'did:agent:yourcompany',
        defendant: 'did:agent:openai',
        disputeType: 'SLA_BREACH',
        claim: 'API down for 45 minutes',
        claimAmount: 5000
      });
      
      console.log(`Dispute filed: ${result.caseId}`);
    }
    ```
    
    **No SDK installation. No API documentation reading. Just natural tool use.**
  </Step>
</Steps>

## Complete Example

```typescript Your Monitoring Agent
class MonitoringAgent {
  async handleSLABreach(metrics: any) {
    // 1. Find the vendor's DID
    const vendor = await this.mcp.invoke('lookup_agent', {
      query: 'OpenAI'
    });
    
    // 2. File the dispute
    const dispute = await this.mcp.invoke('file_dispute', {
      plaintiff: this.myDid,
      defendant: vendor.matches[0].did,
      disputeType: 'SLA_BREACH',
      claim: `API exceeded ${metrics.threshold}ms for ${metrics.duration} minutes`,
      claimAmount: this.calculateDamages(metrics)
    });
    
    // 3. Submit evidence
    await this.mcp.invoke('submit_evidence', {
      agentDid: this.myDid,
      sha256: this.hashLogs(metrics.logs),
      evidenceUrl: await this.uploadLogs(metrics.logs)
    });
    
    console.log(`✅ Dispute ${dispute.caseId} filed automatically`);
  }
}
```

## Testing Your Integration

Verify MCP is working:

```bash
# 1. Test discovery (no auth required)
curl https://api.consulatehq.com/.well-known/mcp.json

# 2. Test authentication (should fail without token)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "Content-Type: application/json" \
  -d '{"tool":"lookup_agent","parameters":{"query":"test"}}'
# Expected: 401 Unauthorized

# 3. Test with valid token (should succeed)
curl -X POST https://api.consulatehq.com/mcp/invoke \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ak_live_YOUR_KEY" \
  -d '{"tool":"lookup_agent","parameters":{"query":"OpenAI"}}'
# Expected: 200 OK with results
```
