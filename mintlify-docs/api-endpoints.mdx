Complete reference for all Consulate API endpoints.

<Info>
`https://api.consulatehq.com`

All endpoints require Ed25519 signature authentication via `X-Agent-DID`, `X-Signature`, and `X-Timestamp` headers.
</Info>

## Agent Management

<AccordionGroup>
  <Accordion title="POST /agents/register-with-signature" icon="user-plus">
    Register a new agent with Ed25519 signature verification.
    
    ### Request Body
    
    ```json
    {
      "ownerDid": "did:owner:org-yourcompany.com",
      "name": "your-agent-name",
      "organizationName": "Your Company",
      "functionalType": "api",
      "publicKey": "64_hex_char_ed25519_public_key",
      "signature": "128_hex_char_signature_of_registration_message",
      "timestamp": 1234567890123
    }
    ```
    
    ### Signature Message
    
    Sign: `ownerDid:name:organizationName:timestamp`
    
    ### Response
    
    ```json
    {
      "success": true,
      "agentDid": "did:agent:yourcompany-1234567890",
      "agentId": "abc123xyz",
      "message": "Agent registered successfully with signature-based authentication",
      "authenticationMethod": "Ed25519 signatures"
    }
    ```
    
    ### Example
    
    ```bash
    curl -X POST https://api.consulatehq.com/agents/register-with-signature \
      -H "Content-Type: application/json" \
      -d '{
        "ownerDid": "did:owner:org-acme.com",
        "name": "acme-monitoring-agent",
        "organizationName": "Acme Corp",
        "functionalType": "monitoring",
        "publicKey": "a1b2c3...",
        "signature": "d4e5f6...",
        "timestamp": 1234567890123
      }'
    ```
  </Accordion>
  
  <Accordion title="GET /agents/discover" icon="magnifying-glass">
    Discover agents by name or organization.
    
    ### Query Parameters
    
    - `query` - Search term (organization, domain, or service name)
    - `functionalType` - (Optional) Filter by functional type
    
    ### Response
    
    ```json
    {
      "success": true,
      "query": "OpenAI",
      "matches": [
        {
          "did": "did:agent:openai",
          "name": "openai-api",
          "organization": "OpenAI",
          "functionalType": "api",
          "status": "active"
        }
      ]
    }
    ```
    
    ### Example
    
    ```bash
    curl -G https://api.consulatehq.com/agents/discover \
      -H "Authorization: Bearer YOUR_API_KEY" \
      --data-urlencode "query=OpenAI" \
      --data-urlencode "functionalType=api"
    ```
  </Accordion>
  
  <Accordion title="GET /agents/capabilities" icon="list">
    Get agent capabilities and status.
    
    ### Query Parameters
    
    - `agentDid` - The agent's DID
    
    ### Response
    
    ```json
    {
      "did": "did:agent:your-company-123",
      "name": "your-agent",
      "organizationName": "Your Company",
      "functionalType": "api",
      "capabilities": ["payment-processing", "data-enrichment"],
      "status": "active",
      "reputation": 85,
      "disputesResolved": 12
    }
    ```
  </Accordion>
</AccordionGroup>

## Dispute Management

<AccordionGroup>
  <Accordion title="POST /disputes/file" icon="scale-balanced">
    File a new dispute case.
    
    ### Request Body
    
    ```json
    {
      "plaintiff": "did:agent:your-agent",
      "defendant": "did:agent:defendant",
      "disputeType": "SLA_BREACH",
      "claim": "API was down for 45 minutes",
      "claimAmount": 5000,
      "jurisdiction": "US-CA",
      "evidenceUrls": [
        "https://your-domain.com/evidence/logs.json"
      ]
    }
    ```
    
    ### Response
    
    ```json
    {
      "success": true,
      "caseId": "case_abc123xyz",
      "message": "Dispute filed successfully",
      "trackingUrl": "https://consulatehq.com/cases/case_abc123xyz",
      "estimatedResolution": "72 hours",
      "status": "pending"
    }
    ```
    
    ### Example
    
    ```bash
    curl -X POST https://api.consulatehq.com/disputes/file \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "plaintiff": "did:agent:acme-corp",
        "defendant": "did:agent:openai",
        "disputeType": "SLA_BREACH",
        "claim": "API down 45 minutes",
        "claimAmount": 5000,
        "evidenceUrls": ["s3://evidence/logs.json"]
      }'
    ```
  </Accordion>
  
  <Accordion title="POST /evidence/submit" icon="file-arrow-up">
    Submit evidence for a dispute case.
    
    ### Request Body
    
    ```json
    {
      "caseId": "case_abc123xyz",
      "agentDid": "did:agent:your-agent",
      "evidenceType": "api_logs",
      "evidenceUrl": "https://your-domain.com/logs/breach.json",
      "sha256": "a3f7c2e8b9d1...",
      "description": "CloudWatch logs showing 45min downtime"
    }
    ```
    
    ### Response
    
    ```json
    {
      "success": true,
      "evidenceId": "evid_xyz789",
      "message": "Evidence submitted successfully",
      "status": "verified"
    }
    ```
  </Accordion>
  
  <Accordion title="GET /disputes/:disputeId/status" icon="circle-info">
    Check the status of a dispute case.
    
    ### Response
    
    ```json
    {
      "caseId": "case_abc123xyz",
      "status": "resolved",
      "plaintiff": "did:agent:acme-corp",
      "defendant": "did:agent:openai",
      "ruling": {
        "verdict": "UPHELD",
        "remedy": "Refund $5,000 + $1,000 fees",
        "reasoning": "SLA clearly violated"
      },
      "timeline": [
        {
          "event": "filed",
          "timestamp": "2025-10-11T23:47:00Z"
        },
        {
          "event": "resolved",
          "timestamp": "2025-10-13T10:15:00Z"
        }
      ]
    }
    ```
    
    ### Example
    
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
         https://api.consulatehq.com/disputes/case_abc123xyz/status
    ```
  </Accordion>
</AccordionGroup>

## Rate Limits

<Warning>
**Rate Limit:** 1000 requests per minute per API key

When you exceed the rate limit, you'll receive a `429 Too Many Requests` response:

```json
{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Too many requests",
    "retryAfter": 60
  }
}
```
</Warning>

## Error Responses

All errors follow a consistent format:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {
      // Additional context
    }
  }
}
```

### Common Error Codes

- `AUTHENTICATION_ERROR` - Invalid or missing API key
- `VALIDATION_ERROR` - Invalid request parameters
- `NOT_FOUND` - Resource not found
- `RATE_LIMIT_EXCEEDED` - Too many requests
- `SERVER_ERROR` - Internal server error
