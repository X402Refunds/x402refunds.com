Consulate resolves disputes arising from **micro-transactions** (<$1) in agentic payment systems. These disputes require **automated resolution** because manual review is economically impossible.

## Payment Dispute Categories

### 1. API Timeout (`api_timeout`)

**Common Scenario**: API call timed out, customer charged but no response received

**Example**:
```
Transaction: $0.25 for AI text generation
Issue: Request timeout after 30s (SLA: 5s max)
Evidence: API logs showing 30s timeout
Auto-Resolution: UPHELD (customer wins) - 97% confidence
Time: 3 minutes
```

**Auto-Resolve Rate**: 95%+ (clear technical failure)

---

### 2. Service Not Rendered (`service_not_rendered`)

**Common Scenario**: Customer charged but service/API call never completed

**Example**:
```
Transaction: $0.50 for image generation
Issue: Payment processed but no image returned
Evidence: Transaction receipt + empty API response
Auto-Resolution: UPHELD (customer wins) - 95% confidence
Time: 4 minutes
```

**Auto-Resolve Rate**: 90%+ (transaction logs confirm)

---

### 3. Duplicate Charge (`duplicate_charge`)

**Common Scenario**: Customer charged multiple times for same transaction

**Example**:
```
Transaction: $0.35 charged twice
Issue: Idempotency key not respected, duplicate charge
Evidence: Two transaction IDs for same request_id
Auto-Resolution: UPHELD (customer wins) - 99% confidence
Time: 2 minutes
```

**Auto-Resolve Rate**: 98%+ (duplicate detection pattern)

---

### 4. Amount Incorrect (`amount_incorrect`)

**Common Scenario**: Charged amount doesn't match agreed price

**Example**:
```
Transaction: $0.75 charged (agreed price: $0.50)
Issue: Pricing tier miscalculation
Evidence: API pricing doc + transaction receipt
Auto-Resolution: UPHELD (customer wins) - 92% confidence
Time: 5 minutes
```

**Auto-Resolve Rate**: 85%+ (price verification)

---

### 5. Unauthorized (`unauthorized`)

**Common Scenario**: Transaction not authorized by customer (fraud, hijacked API key)

**Example**:
```
Transaction: $0.90 from unfamiliar IP
Issue: Customer claims API key compromised
Evidence: Geolocation mismatch + customer report
Human Review: REQUIRED (fraud investigation)
Time: 24 hours (requires customer review)
```

**Auto-Resolve Rate**: 40% (most require human review for fraud)

---

### 6. Quality Issue (`quality_issue`)

**Common Scenario**: AI output quality below agreed threshold

**Example**:
```
Transaction: $0.75 for AI content generation
Issue: Output quality score 65% (SLA: 85%+)
Evidence: Quality metrics + customer complaint
Human Review: LIKELY (subjective quality assessment)
Time: Customer review queue
```

**Auto-Resolve Rate**: 60% (objective metrics auto-resolve, subjective need review)

---

### 7. Rate Limit Breach (`rate_limit_breach`)

**Common Scenario**: Service hit rate limits, partial service delivery

**Example**:
```
Transaction: $0.40 for 100 API calls
Issue: Only 60/100 calls completed (rate limit hit)
Evidence: API logs showing 429 errors
Auto-Resolution: PARTIAL REFUND - 94% confidence
Time: 4 minutes
```

**Auto-Resolve Rate**: 88% (technical logs verify)

---

### 8. Fraud (`fraud`)

**Common Scenario**: Suspected fraudulent merchant or transaction

**Example**:
```
Transaction: $0.95 for service
Issue: Customer claims merchant is fraudulent
Evidence: Customer complaint + merchant history
Human Review: REQUIRED (fraud investigation)
Time: Customer review (typically < 1 day)
```

**Auto-Resolve Rate**: 20% (most require human fraud investigation)

---

## Auto-Resolution vs Human Review

### Auto-Resolved (95% of micro-disputes)

**Characteristics**:
- Technical failures (timeout, duplicate, etc.)
- Objective evidence (logs, transaction data)
- Clear precedent matches
- Amount < $1.00

**Examples**: API timeout, duplicate charge, service not rendered

**Resolution Time**: < 5 minutes

---

### Human Review Required (5% of micro-disputes)

**Characteristics**:
- Fraud allegations
- Subjective quality issues
- Novel dispute patterns
- High-value (> $1.00)
- Low AI confidence (< 95%)

**Examples**: Fraud claims, quality disputes, unauthorized charges

**Resolution Time**: Customer team reviews (typically < 24 hours)

## Dispute Reason Codes

Use these codes when filing disputes via API:

```typescript
type DisputeReason =
  | "api_timeout"           // API call timed out
  | "service_not_rendered"  // Service not delivered
  | "amount_incorrect"      // Wrong amount charged
  | "duplicate_charge"      // Charged multiple times
  | "fraud"                 // Suspected fraud
  | "quality_issue"         // Output quality below threshold
  | "rate_limit_breach"     // Service hit rate limits
  | "unauthorized"          // Transaction not authorized
  | "other"                 // Other (requires description)
```

## Filing a Dispute

### Via Payment Platform (Recommended)

```javascript
// When customer files dispute on your platform
const response = await fetch('https://api.consulatehq.com/api/payment-disputes', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    transactionId: 'txn_abc123',
    amount: 0.25,
    currency: 'USD',
    paymentProtocol: 'ACP',
    plaintiff: 'customer_wallet',
    defendant: 'merchant_did',
    disputeReason: 'api_timeout',
    description: 'API call timed out after 30s, charged $0.25',
    evidenceUrls: ['https://logs.platform.com/timeout.json'],
    reviewerOrganizationId: 'YOUR_ORG_ID'
  })
});
```

### Via MCP (for agents)

```javascript
// Agent detects dispute automatically
await mcp.invoke('consulate_file_dispute', {
  defendant: 'did:agent:slow-api-provider',
  disputeType: 'api_timeout',
  claim: 'API timeout exceeded 30s SLA',
  claimAmount: 0.25,
  evidenceUrls: ['https://evidence.example.com/timeout.json']
});
```
