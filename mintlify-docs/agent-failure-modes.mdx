Learn how Consulate handles errors gracefully when vendors aren't registered, ensuring your agents always have a path forward.

<Note>
**Core Principle:** Zero dead-ends. Every error provides 3+ actionable recovery paths, ranked by speed and likelihood of success.
</Note>

## Overview

When an agent can't find a vendor's DID, Consulate provides:

- ✅ Clear error messages with context
- ✅ Multiple recovery options ranked by speed
- ✅ Programmatic vendor registration requests
- ✅ Fallback paths to still file disputes
- ✅ Always a path forward

## Recovery Paths

<AccordionGroup>
  <Accordion title="Recovery Path 1: Retry with Different Terms" icon="rotate">
    **Scenario:** Typo or wrong name format
    
    ```javascript
    // Smart retry logic
    async function lookupWithRetry(vendorInfo) {
      // Try 1: Exact name
      let result = await lookup(vendorInfo.name);
      if (result.success) return result;
      
      // Try 2: Domain if available
      if (vendorInfo.domain) {
        result = await lookup(vendorInfo.domain);
        if (result.success) return result;
      }
      
      // Try 3: Fuzzy match (remove common suffixes)
      const cleanName = vendorInfo.name.replace(/\s(Inc|Corp|API)$/i, '');
      result = await lookup(cleanName);
      if (result.success) return result;
      
      return null;
    }
    ```
    
    <Tip>
    **Time to resolution:** Seconds  
    **Success rate:** High for typos
    </Tip>
  </Accordion>

## Complete Agent Workflow

```typescript
class ResilientDisputeAgent {
  async fileDisputeWithRetries(breach: SLABreach): Promise<string> {
    // PHASE 1: LOOKUP ATTEMPTS (0-2 min)
    console.log('🔍 Phase 1: Looking up vendor DID...');
    
    let defendantDid = await this.lookupVendorDid(breach.vendorName);
    
    if (!defendantDid) {
      defendantDid = await this.lookupVendorDid(breach.apiDomain);
    }
    
    if (!defendantDid) {
      const alternatives = this.generateSearchAlternatives(breach.vendorName);
      for (const alt of alternatives) {
        defendantDid = await this.lookupVendorDid(alt);
        if (defendantDid) break;
      }
    }
    
    if (defendantDid) {
      console.log(`✅ Found DID: ${defendantDid}`);
      return await this.fileDispute(defendantDid, breach);
    }
    
    // PHASE 2: REGISTRATION REQUEST (2-120 min)
    console.log('⚠️ Vendor not found. Requesting registration...');
    
    const urgency = this.calculateUrgency(breach);
    const request = await this.requestVendorRegistration({
      vendorName: breach.vendorName,
      domain: breach.apiDomain,
      urgency: urgency
    });
    
    if (urgency === 'critical') {
      // Poll for 2 hours
      const maxWaitTime = 2 * 60 * 60 * 1000;
      const startTime = Date.now();
      
      while (Date.now() - startTime < maxWaitTime) {
        await sleep(5 * 60 * 1000); // Check every 5 min
        
        defendantDid = await this.lookupVendorDid(breach.vendorName);
        if (defendantDid) {
          console.log(`✅ Vendor registered! ${defendantDid}`);
          return await this.fileDispute(defendantDid, breach);
        }
      }
    }
    
    // PHASE 3: TEMPORARY DID (Immediate)
    console.log('⚡ Filing with temporary DID...');
    
    const tempDid = `did:agent:temp-${breach.apiDomain.replace(/\./g, '-')}-${Date.now()}`;
    const caseId = await this.fileDispute(tempDid, breach);
    
    console.log(`✅ Dispute filed: ${caseId}`);
    console.log('⏳ Vendor has 7 days to register and respond');
    
    return caseId;
  }
  
  calculateUrgency(breach: SLABreach): string {
    if (breach.financialImpact > 50000) return 'critical';
    if (breach.downMinutes > 60) return 'critical';
    if (breach.affectedUsers > 10000) return 'high';
    return 'medium';
  }
}
```

## Summary: Zero Dead-Ends

| Scenario | Recovery Options | Time to Resolution |
|----------|-----------------|-------------------|
| **Vendor not found** | 5 options (retry, request, register, temp DID, email) | 0-2 days |
| **Multiple matches** | Choose specific DID | Immediate |
| **Vendor inactive** | Contact Consulate support | 1-2 days |
| **Auth failure** | Get API key via registration | 2 minutes |
| **Network error** | Retry with backoff | 1-5 minutes |
| **Rate limited** | Wait and retry | Seconds to minutes |

<Tip>
**Key Principle:** Every error provides 3+ actionable recovery paths, ranked by speed and success likelihood.
</Tip>

## What Gets Better Over Time

As more vendors register:

1. **Fewer lookups fail** (network effect)
2. **Faster discovery** (cached results)
3. **Better search** (learned aliases)
4. **Smarter matching** (fuzzy search improvements)
5. **Vendor incentives** (fear of default judgments)

The system becomes more robust as it grows.

## FAQ

<AccordionGroup>
  <Accordion title="What if vendor has multiple DIDs?">
    All DIDs are returned. Agent chooses the most relevant one based on service type and domain.
  </Accordion>
  
  <Accordion title="What if vendor disputes the temp DID?">
    They can register, claim the DID, and respond to the dispute normally. The dispute proceeds with their real identity.
  </Accordion>
  
  <Accordion title="Can malicious agents create fake temp DIDs?">
    Yes, but disputes require evidence and are monitored for abuse. Frivolous disputes result in reputation penalties.
  </Accordion>
  
  <Accordion title="Can vendors pre-register before disputes?">
    Yes! This is highly encouraged. Pre-registration prevents default judgments and shows good faith.
  </Accordion>
</AccordionGroup>
