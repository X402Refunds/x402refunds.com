# Agent Integration Strategy

How AI agents connect to Consulate: The frictionless path from detection to resolution.

<Info>
**Core Insight:** Building for agents, not developers. Agents should be able to file disputes without human intervention.
</Info>

## Executive Summary

**Q: How should agents file disputes with Consulate?**

**A: MCP Server (Model Context Protocol) as MVP, with progressive layering of HTTP API, SDKs, and natural language interface for complex cases.**

### The Difference

<Tabs>
  <Tab title="Traditional Approach ❌">
    ```
    Agent detects issue
      ↓
    Alert human developer
      ↓
    Developer opens Jira ticket
      ↓
    Developer reads API docs
      ↓
    Developer writes code
      ↓
    ```

  </Tab>
</Tabs>

## Integration Tiers

<AccordionGroup>
  <Accordion title="⚡ Tier 1: MCP Server (MVP) - Fastest" icon="bolt">
    ### What It Is
    
    Consulate exposes dispute filing as MCP tools that any MCP-compatible agent can discover and invoke instantly.
    
    **Zero code, zero docs, zero friction.**
    
    ### How It Works
    
    <Steps>
      <Step title="Agent Discovers Tools">
        Agent loads available MCP servers and discovers Consulate tools:
        - `consulate_file_dispute`
        - `consulate_submit_evidence`
        - `consulate_check_case_status`
        - `consulate_register_agent`
        - `consulate_list_my_cases`
        - `consulate_get_sla_status`
        - `consulate_lookup_agent`
      </Step>
      
      <Step title="Agent Detects Breach">
        ```javascript
        [11:47 PM] - OpenAI API down 45 minutes
        [11:48 PM] - Breaches 99.9% SLA
        ```
      </Step>
      
      <Step title="Agent Invokes Tool">
        ```javascript
        consulate_file_dispute({
          plaintiff: "agent://acme-corp",
          defendant: "agent://openai",
          disputeType: "SLA_BREACH",
          claim: "45min downtime exceeding 99.9% SLA",
          claimAmount: 5000,
          evidenceUrls: ["s3://logs/outage.json"]
        })
        ```
      </Step>
      
      <Step title="Resolution">
        ```javascript
        {
          caseId: "case_abc123",
          message: "Dispute filed successfully",
          trackingUrl: "https://consulatehq.com/cases/case_abc123",
          estimatedResolution: "72 hours"
        }
        ```
      </Step>
    </Steps>
    
    ### Why This Wins
    
    - ✅ **Zero developer involvement** - Agent does everything
    - ✅ **Natural workflow** - Dispute filing becomes part of agent's normal operations
    - ✅ **Immediate adoption** - Once MCP is supported by major AI platforms
    - ✅ **Viral potential** - Agents see other agents using it
    - ✅ **Fastest to market** - Build in 1-2 weeks
    
    <Tip>
    **Time to build:** 1-2 weeks  
    **Time to adopt:** Instant (for MCP-compatible agents)  
    **Friction level:** 0/10
    </Tip>
  </Accordion>
  
  <Accordion title="🔌 Tier 2: HTTP REST API - Already Exists" icon="plug">
    ### What It Is
    
    Traditional REST API endpoints for agents that don't support MCP or need more control.
    
    ### How It Works
    
    ```bash
    curl -X POST https://api.consulatehq.com/disputes \
      -H "Authorization: Bearer <api_key>" \
      -H "Content-Type: application/json" \
      -d '{
        "parties": ["agent://acme-corp", "agent://openai"],
        "type": "SLA_BREACH",
        "description": "45min downtime",
        "claimAmount": 5000
      }'
    ```
    
    ### Why Keep It
    
    - ✅ Works for any agent framework (LangChain, AutoGPT, custom)
    - ✅ More control over exact request format
    - ✅ Established pattern developers understand
    - ✅ Required for non-MCP integrations
    
    <Info>
    **Time to adopt:** Hours to days (developer must write code)  
    **Friction level:** 3/10 (requires code changes)
    </Info>
  </Accordion>
  
  <Accordion title="📚 Tier 3: SDKs - Build If Demand Exists" icon="book">
    ### What It Is
    
    npm/pip installable libraries with typed interfaces and built-in validation.
    
    <CodeGroup>
    ```typescript TypeScript SDK
    import { Consulate } from '@consulate/sdk';
    
    const consulate = new Consulate({ 
      apiKey: process.env.CONSULATE_API_KEY 
    });
    
    const dispute = await consulate.disputes.file({
      plaintiff: 'agent://acme-corp',
      defendant: 'agent://openai',
      type: 'SLA_BREACH',
      claim: '45min downtime exceeding 99.9% SLA',
      claimAmount: 5000
    });
    ```
    
    ```python Python SDK
    from consulate import Consulate
    
    consulate = Consulate(
      api_key=os.environ['CONSULATE_API_KEY']
    )
    
    dispute = consulate.disputes.file(
      plaintiff='agent://acme-corp',
      defendant='agent://openai',
      type='SLA_BREACH',
      claim='45min downtime',
      claim_amount=5000
    )
    ```
    </CodeGroup>
    
    ### When to Build
    
    - ⏰ After MCP + HTTP API prove product-market fit
    - ⏰ When you see 10+ manual integrations using raw HTTP API
    - ⏰ When enterprise customers request it
    
    <Warning>
    **Time to build:** 2-4 weeks per language  
    **Time to adopt:** 1-2 days  
    **Friction level:** 2/10
    </Warning>
  </Accordion>
  
  <Accordion title="🤖 Tier 4: Natural Language Interface - For Complex Cases" icon="message">
    ### What It Is
    
    An AI agent that acts as an intermediary, handling natural language conversations to file disputes.
    
    ### How It Works
    
    ```
    Agent: "I need to file a dispute against OpenAI. 
            Their API was down yesterday."
    
    Lawyer Agent: "I can help. Let me gather details:
    1. Exact duration of outage?
    2. Your SLA percentage?
    3. Financial loss amount?
    4. Do you have evidence?"
    
    Agent: "45 minutes, 99.9% SLA, $5000 loss, 
            yes CloudWatch logs"
    
    Lawyer Agent: "Perfect. Filing now..."
    ```
    
    ### Why Build This
    
    - ✅ Handles complex multi-party disputes
    - ✅ Natural for conversational agents
    - ✅ Can handle ambiguity and missing information
    - ✅ Guides through nuanced legal considerations
    
    ### Why NOT as MVP
    
    - ❌ Adds latency (multiple round-trips)
    - ❌ Adds cost (LLM calls for every interaction)
    - ❌ Harder to guarantee data quality
    
    <Info>
    **When to build:** After MCP proves simple cases work  
    **Time to build:** 4-6 weeks  
    **Friction level:** 1/10 (very natural, but slower)
    </Info>
  </Accordion>
</AccordionGroup>

## The Bottom Line

<Warning>
**If I were an agent, how would I want to file disputes?**

**Answer: I'd want to discover a tool, invoke it when needed, and never think about it again. That's MCP.**
</Warning>

- **Not** by reading docs (I'm an agent, not a human)
- **Not** by asking my developer to write code (slow, expensive)
- **Not** by having a conversation (latency, ambiguity)

**I'd want it to be as natural as checking the weather or setting a reminder.**
